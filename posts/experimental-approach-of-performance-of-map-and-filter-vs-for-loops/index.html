#Experimental approach of Performance of map(), filter() and reduce() vs for-loops in Swift

##Introduction: ⋅⋅⋅In Swift programming there is a paradigm called functional programming Map, filter and reduce are examples of this paradigm. In swift, we use these functions to loop over collections without using a for-loop.

##The Basic Setup:

###Hardware: The data shown here was all generated from my personal Macbook Pro (macOS Catalina v10.15.6, 2.5 GHz Quad-Core Intel Core i5 with16 GBs 600 MHz DDR3)

###Basic Setup In terms of software, I used Xcode v11.3.1 with the default Swift v5.1.3. For the purpose of comparisons, I created a Project with Unit Tests. I wrote many unit tests to measure the performance of each function. For the two functions, I ran one pair of tests 3 times to compare the results of order functions (map, filter, reduce) and loops. Each unit test used an array with 10000000 of integer numbers. The code for running these experiments can be found on GitHub.

##Experiments: ###Map vs Loop In this experiment, the test converts the first 1000000 to its square value. The performance of normal for-loop is roughly double the map function

Test Map Loop
Test #1 2.792 5.213
Test #2 2.690 3.943
Test #3 2.466 4.142

###Filter vs Loop In this experiment, the test filters the even values of the first 1000000 values. The performance of normal for-loop is greater than filter function.

Test Map Loop
Test #1 2.120 2.996
Test #2 2.15 3.275

###Reduce vs Loop In this experiment, the test obtains the sum of the first 1000000 values. The performance of normal for-loop is less than the reduce function.

Test Map Loop
Test #1 2.087 1.732
Test #2 2.477 1.664

##Conclusions: …Map and Filter are more efficient than loops with large data, but the difference isn’t going to be significant when used with smaller than 1000000 values. However, it is a good idea to know about functional programming, because it is used in reactive programming.
